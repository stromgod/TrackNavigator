GpxHelper.java — это утилитный файл для работы с GPX (формат GPS-треков): он сохраняет и читает точки маршрута.
Основные задачи GpxHelper:

Запись трека в GPX (writeTrack(...))
формирует XML GPX 1.1 (<gpx>, <trk>, <trkseg>, <trkpt>);
записывает список LatLngPoint как точки трека (lat, lon);
добавляет время для точек в UTC (ISO-8601);
экранирует спецсимволы в имени трека (escapeXml).

Чтение точек из GPX (readPoints(...))
парсит входной GPX через XmlPullParser;
собирает все точки trkpt (точки трека) в порядке документа;
если trkpt нет, использует wpt (waypoints) как запасной вариант;
возвращает List<LatLngPoint>.

Безопасный парсинг координат (parsePoint(...))
извлекает атрибуты lat и lon;
конвертирует их в double;
пропускает битые/неполные точки (возвращает null).
Итого: файл отвечает за “перевод” между внутренними точками приложения и GPX-файлом.



LatLngPoint.java — это простой класс-точка с координатами.

Что он делает:

хранит широту (latitude) и долготу (longitude);
поля final, то есть объект неизменяемый после создания;
используется как единый тип координат в проекте (например, в GpxHelper, RaceActivity, GeoUtils).
По сути, это базовый “контейнер” для GPS-точки, чтобы передавать координаты между модулями без лишней логики.


GeoUtils.java — это математический модуль для геометрии маршрута по GPS-координатам.

Что делает файл
Он дает функции, которые нужны, чтобы:

посчитать расстояния между точками в метрах;
понять, насколько пользователь отклонился от текущего сегмента маршрута;
определить, с какой стороны от линии маршрута находится пользователь (слева/справа).
Его основные задачи
Перевод координат в локальную плоскость ENU
toEnMeters(...) переводит lat/lon в локальные метры (east/north) относительно опорной точки.
Это упрощает линейную геометрию на небольшом участке трека.

Расчет отклонения от сегмента
distancePointToSegmentMeters(a, b, p) находит ближайшую точку на отрезке a-b к точке p и возвращает расстояние до нее в метрах.
Это и есть ключевая метрика “насколько сошел с маршрута”.

Определение стороны отклонения
crossTrackSign(a, b, p) через векторное произведение определяет знак:

> 0 — точка слева от направления a -> b,
< 0 — справа.
Точное сферическое расстояние
distanceMeters(a, b) считает расстояние по формуле Хаверсина (по сфере Земли), чтобы метры были корректными.


MainActivity.java — это главный экран-меню приложения (точка входа).

Что он делает:

загружает layout activity_main;
находит 3 кнопки: btnRecord, btnRace, btnExit;
по нажатию:
btnRecord открывает RecordTrackActivity (запись трека),
btnRace открывает RaceActivity (заезд по треку),
btnExit закрывает приложение через finishAffinity().
То есть MainActivity не содержит навигационной логики — он только маршрутизирует пользователя в нужный модуль.



RaceActivity.java — это экран “заезда” по загруженному GPX-треку: он получает GPS-позицию, сравнивает её с текущим сегментом маршрута и показывает, на линии ты или отклонился влево/вправо.

Как он работает по шагам:

Инициализация (onCreate)

Загружает activity_race.
Привязывает UI-элементы (textCheckpoint, textCoords, textAccuracy, textDeviation, кнопки).
Назначает обработчики кнопок: выбрать GPX, старт, стоп, назад.
Загрузка GPX

pickGpx() открывает системный выбор файла.
loadGpxFromUri() читает файл через GpxHelper.readPoints(...).
Если точек меньше 2 — старт блокируется.
Если всё ок — активируется кнопка старта и показывается прогресс чекпоинтов.
Старт заезда (startRace)

Проверяет, что трек загружен.
Проверяет разрешения геолокации.
Проверяет включен ли GPS.
Подписывается на GPS-обновления (requestLocationUpdates).
Сбрасывает индекс сегмента на начало, включает режим гонки, запускает периодический tickRunnable.
Приём GPS (locationListener)

Сохраняет последнюю точку lastFix.
Пока гонка активна — обновляет координаты и точность на экране.
Периодическая проверка положения (evaluatePosition)

Если GPS-фикс отсутствует — статус “ожидание GPS”.
Если точность плохая (> MAX_FIX_ACCURACY_M, у тебя 8 м) — не считает отклонение.
Определяет текущий прогресс по маршруту:
пока ты близко к следующей вершине (PASS_CHECKPOINT_M, 3 м), двигает segmentStartIndex.
Если дошёл до конца — completeRace().
Иначе берёт текущий сегмент a-b и считает поперечное отклонение:
GeoUtils.distancePointToSegmentMeters(a,b,p).
Если отклонение <= DEVIATION_THRESHOLD_M (5.5 м) — “на маршруте”.
Если больше — через GeoUtils.crossTrackSign(...) определяет сторону:
>0 влево, <0 вправо, и подсвечивает предупреждением.
Завершение/остановка

completeRace() — останавливает обновления, показывает “финиш”, возвращает на MainActivity.
stopRaceUser() — ручной стоп: отключает GPS-апдейты, сбрасывает состояние UI.
onDestroy() тоже вызывает остановку, чтобы не оставить подписку на GPS.

ActivityResultLauncher<Intent> — получить файл GPX от системного файлового пикера.

Ты запускаешь Intent (выбор документа),
пользователь выбирает файл,
результат приходит в callback (URI файла),
дальше вызывается loadGpxFromUri(uri).
Это современная замена старого startActivityForResult.
LocationListener — получать GPS-обновления в реальном времени.

Его передают в locationManager.requestLocationUpdates(...),
при каждом новом фиксe вызывается onLocationChanged(...),
в нем сохраняется lastFix и обновляются координаты/точность на экране,
на основе lastFix потом evaluatePosition() вычисляет отклонение от трека.
Коротко:
ActivityResultLauncher = входные данные маршрута (GPX-файл),
LocationListener = текущая позиция пользователя (GPS).


RecordTrackActivity — это модуль записи маршрута: собирает GPS-точки вручную и сохраняет их в GPX-файл.

Что делает файл по шагам:

Запускает GPS и показывает статус

проверяет разрешения геолокации;
включает обновления LocationManager;
отображает текущие координаты и точность (textCoords, textAccuracy).
Контролирует качество точек

точка добавляется только если есть lastFix и точность достаточная:
порог: MAX_ACCURACY_METERS = 7f.
если точность хуже — показывает предупреждение и не добавляет точку.
Добавляет контрольные точки маршрута

по кнопке btnAddPoint берет текущие координаты;
сохраняет их как LatLngPoint в список controlPoints;
обновляет счётчик точек на экране.
Сохраняет маршрут в GPX

по btnFinish проверяет минимум 2 точки;
спрашивает имя файла;
предлагает 2 режима сохранения:
вручную через системный выбор места (CreateDocument);
автоматически в папку Documents.
записывает GPX через GpxHelper.writeTrack(...).
Аккуратно завершает работу

после успешного сохранения останавливает GPS (stopGps) и закрывает экран;
также останавливает GPS в onDestroy, чтобы не было утечки подписки.


URI (Uniform Resource Identifier) — это строка-идентификатор ресурса.

В Android это обычно “ссылка” на файл/данные, но не обязательно прямой путь на диск.

Примеры:

веб: https://site.com/file.gpx
файл через провайдер Android: content://com.android.providers.../1234
локальный файл (реже): file:///storage/.../track.gpx
Зачем это нужно в твоем проекте:

когда пользователь выбирает GPX через системный пикер, ты получаешь Uri;
по этому Uri открываешь поток (openInputStream / openOutputStream) и читаешь/пишешь файл;
так Android безопасно дает доступ к данным между приложениями.
Коротко: URI — это адрес-идентификатор ресурса, с которым приложение работает через ContentResolver.
